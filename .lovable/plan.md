## إعادة تصميم لوحة تحكم الإدارة (نسخة الجوال فقط)

### النطاق
- التعديل يطبّق فقط على شاشات الجوال (`<768px`) عبر `useIsMobile()` وclassaات Tailwind `md:`.
- لا تغيير على نسخة سطح المكتب (كل ما هو موجود يبقى كما هو داخل شرط `!isMobile` أو classات `hidden md:block`).
- ملف العمل الأساسي: `src/components/rifans/AdminDashboard.tsx` (3223 سطر).

### 1. تخطيط جديد للجوال (Mobile Shell)
- إنشاء غلاف مكوّن داخلي `MobileAdminShell` يُعرض فقط عند `isMobile`.
- الشكل العام:
  - Header علوي مضغوط (شعار + إشعارات + خروج).
  - محتوى التبويب النشط (Full width, padding مريح).
  - Bottom Tab Bar ثابت (5 تبويبات).
- ارتفاع الشاشة `100dvh`، تمرير عمودي فقط، إلغاء أي `overflow-x`.

### 2. التبويبات الخمسة (Bottom Nav)
1. الرئيسية (`home`)
2. الطلبات (`requests`)
3. العملاء (`clients`)
4. المستندات (`documents`)
5. المزيد (`more`)

كل تبويب يستدعي بيانات موجودة أصلاً من نفس الـstate الحالي — لا تغيير في منطق الجلب/الـAPI.

### 3. الصفحة الرئيسية (Home) للجوال
تعرض فقط:
- **ملخص اليوم**: بطاقة واحدة (طلبات اليوم + عملاء جدد + مستندات معلّقة).
- **أهم 4 مؤشرات**: شبكة 2×2 من KPI cards (إجمالي الطلبات، قيد المعالجة، مكتملة، إيرادات).
- **آخر النشاطات**: قائمة عمودية بآخر 5 عناصر (اسم العميل + النوع + الوقت).
- **إجراءات سريعة**: أزرار كبيرة (طلب جديد، عرض الطلبات، فتح العملاء، فتح المحادثات).

### 4. تبويب الطلبات — "إدارة الطلبات"
- شريط فلاتر أفقي (chips) بالحالات:
  - جديدة / قيد المعالجة / بانتظار التوقيع / مكتملة / متوقفة.
- تحت الشريط: قائمة بطاقات طلب (Card) بدلاً من الجدول، تعرض:
  - رقم الطلب، اسم العميل، النوع، الحالة (Badge ملوّن)، التاريخ، زر "فتح".

### 5. تبويب العملاء — بطاقات بدل الجدول
- إلغاء الجدول على الجوال (يبقى للـ desktop).
- كل بطاقة عميل تعرض:
  - اسم العميل
  - رقم الملف
  - نوع الطلب (آخر طلب)
  - الحالة (Badge)
  - زر "فتح الملف" يفتح ملف العميل بنفس منطق `setSelectedUser` الحالي.

### 6. تبويب المستندات
عرض قائمة بطاقات للعقود/الفواتير/سندات الأمر الموجودة بالفعل (نفس البيانات) بترتيب زمني، مع فلترة بسيطة (الكل / عقود / فواتير / سندات).

### 7. تبويب المزيد
روابط للأقسام الأقل استخداماً الموجودة حالياً في الديسكتوب:
- الإشعارات، المحادثات، الإعدادات، الإحصائيات المفصّلة، تسجيل الخروج.

### 8. النوافذ المنبثقة (Modals) في الجوال
- تحويل جميع الـmodals الحالية إلى **Full-Screen Sheet** عند `isMobile`:
  - `fixed inset-0`, `h-[100dvh]`, `rounded-none`, header ثابت مع زر رجوع سهم يمين، ومحتوى قابل للتمرير.
- الديسكتوب يبقى modal مركزي كما هو.
- ينطبق على: ملف العميل، تفاصيل الطلب، إنشاء فاتورة، رفع مستند، محادثة… إلخ (نفس المكوّنات، فقط wrapper classes مشروطة بـ`isMobile`).

### 9. الهوية البصرية
- الحفاظ على الألوان الحالية: Dark Purple `#22042C`، Gold `#C7A969`، خط Tajawal.
- تكبير حجم الخط الأساسي على الجوال (14→15px) وزيادة المسافات (`p-4`، `gap-3`).
- الأزرار بحد أدنى `h-11` لسهولة اللمس.
- Badges موحدة الألوان حسب الحالة (أصفر=معلّق، أزرق=معالجة، بنفسجي=توقيع، أخضر=مكتمل، رمادي=متوقف).

### تفاصيل تقنية
- كل التغييرات داخل `AdminDashboard.tsx`؛ إضافة مكونات فرعية داخلية:
  - `MobileAdminShell`, `MobileHome`, `MobileRequests`, `MobileClients`, `MobileDocuments`, `MobileMore`, `MobileBottomNav`, `MobileSheet` (wrapper).
- استخدام `useIsMobile()` (موجود في `src/hooks/use-mobile.tsx`) في أعلى المكوّن:
  ```tsx
  if (isMobile) return <MobileAdminShell ... />;
  // else: التصميم الحالي كما هو بدون تغيير
  ```
- إعادة استخدام نفس الـstate والـfetch functions والـhandlers الحالية — بدون تعديل منطق الأعمال أو الـqueries.
- إبقاء زر تسجيل الخروج في الهيدر وأيضاً في تبويب "المزيد".

### ما لن يتغيّر
- تصميم سطح المكتب حرفياً (JSX الحالي محفوظ داخل فرع `else`).
- منطق قاعدة البيانات، الـedge functions، صلاحيات RLS، والـauth.
- ملفات أخرى غير `AdminDashboard.tsx` (باستثناء استيراد `useIsMobile` إن لم يكن مستورداً).
